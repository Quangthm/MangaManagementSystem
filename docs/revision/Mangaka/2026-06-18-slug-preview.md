# Session Note — Mangaka Dashboard Slug Preview

- **Branch:** `feature/Mangaka`
- **Date:** 2026-06-18

---

## Problem

The Create Draft and Edit Draft modals did not show the user what slug would be generated from the title before saving. Users had no visibility into the URL identity their series would receive.

## Fix Summary

Added a read-only slug preview that shows the user what slug will be generated from the current title before saving. The preview uses the same `SlugGenerator.FromTitle()` logic that the backend uses, ensuring the preview matches the final slug exactly.

The preview is labeled "Preview only — final slug is generated by the server" to make it clear that the UI is not authoritative.

---

## Preview Behavior

### Create Draft Modal
- Title field has `Immediate="true"` for live updates
- Below the Title field:
  - If title is empty: "Slug preview will appear after entering a title."
  - If title has content: "Slug preview: `my-new-manga`" (monospace, indigo color)
  - Note below: "Preview only — final slug is generated by the server."
- Preview updates as the user types (not on blur)

### Edit Draft Modal
- Title field has `Immediate="true"` for live updates
- Same preview behavior as Create Draft modal
- Preview shows immediately when modal opens with pre-filled title
- Preview updates as the user edits the title

### Helper Method
```csharp
private static string BuildSlugPreview(string? title)
{
    if (string.IsNullOrWhiteSpace(title))
        return string.Empty;

    try
    {
        return SlugGenerator.FromTitle(title);
    }
    catch
    {
        return string.Empty;
    }
}
```

The helper wraps `SlugGenerator.FromTitle()` in a try-catch to handle any edge cases gracefully. The backend `SlugGenerator` class is in `MangaManagementSystem.Application.Common` and is accessible from the Web project.
