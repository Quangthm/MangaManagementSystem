# MangaManagementDB - Database Structure

Source: `MangaManagementSystem_Schema.sql`

## Phase 7 GUID Alignment Note

This document follows the current GUID-based schema. Business/domain primary keys and matching foreign keys use SQL Server `UNIQUEIDENTIFIER` with `DEFAULT NEWID()` where generated by the table. Do not describe current business IDs as `INT IDENTITY`, `BIGINT IDENTITY`, or `SMALLINT IDENTITY`.

Allowed exception: `audit.AuditEvent.audit_event_id` may remain `BIGINT IDENTITY` because audit is append-only and chronological ordering is useful.

## Database

- Name: `MangaManagementDB`
- Schemas: `auth`, `manga`, `audit`

## Schemas

| Schema | Responsibility |
|---|---|
| `auth` | Roles, users, login/account state, one MVP role per user. |
| `manga` | Manga workflow data, file metadata, series/proposals, chapters/pages, tasks, rankings, notifications. |
| `audit` | Append-only audit events for security/business traceability. |

## Core Tables

### auth.Roles

| Column | Type | Notes |
|---|---|---|
| `role_id` | `UNIQUEIDENTIFIER` | PK, default `NEWID()` |
| `role_name` | `NVARCHAR(30)` | Required, unique |

Seeded/expected MVP roles:

- `Mangaka`
- `Assistant`
- `Tantou Editor`
- `Editorial Board Member`
- `Editorial Board Chief`
- `Admin`

### auth.Users

| Column | Type | Notes |
|---|---|---|
| `user_id` | `UNIQUEIDENTIFIER` | PK, default `NEWID()` |
| `role_id` | `UNIQUEIDENTIFIER` | FK to `auth.Roles(role_id)` |
| `username` | `NVARCHAR(50)` | Required, unique, login/system identifier |
| `display_name` | `NVARCHAR(100)` | Required, user-facing display name |
| `email` | `NVARCHAR(254)` | Required, unique |
| `password_hash` | `NVARCHAR(255)` | Required, BCrypt hash |
| `avatar_file_id` | `UNIQUEIDENTIFIER` | Nullable FK to `manga.FileResource` |
| `portfolio_file_id` | `UNIQUEIDENTIFIER` | Nullable FK to `manga.FileResource` |
| `status_code` | `NVARCHAR(30)` | Required, default `PENDING_APPROVAL`; allowed `PENDING_APPROVAL`, `REJECTED`, `ACTIVE`, `DISABLED` |
| `created_at_utc` | `DATETIME2(7)` | Required, default `SYSUTCDATETIME()` |

Important behavior:

- Public registration creates users as `PENDING_APPROVAL`.
- Public Admin registration may request role `Admin`, but it must still create only a pending account.
- Rejected and disabled users cannot log in.
- Rejected users keep username/email reserved in MVP.

### Board, Chapter, Page, Region, Task, Ranking, Notification Tables

The remaining manga workflow tables follow the same GUID ID standard:

- `manga.SeriesBoardPoll`
- `manga.SeriesBoardVote`
- `manga.Chapter`
- `manga.ChapterPage`
- `manga.ChapterPageVersion`
- `manga.PageRegion`
- `manga.ChapterPageAnnotation`
- `manga.ChapterPageTask`
- `manga.ChapterPageTaskRegion`
- `manga.ChapterEditorialReview`
- `manga.SeriesRankingSnapshot`
- `manga.ChapterReaderVoteSnapshot`
- `manga.Notification`

Key rules:

- FK columns referencing business/domain rows use `UNIQUEIDENTIFIER`.
- Status/code columns use `NVARCHAR` code values such as `UNDER_REVIEW`, `APPROVED`, `REJECTED`, `CANCELLED`, `ASSIGNED`, and similar domain codes.
- Soft-delete pairs must be either both `NULL` or both non-`NULL`.
- Filtered unique indexes enforce active/current row rules such as one current page version and one active page number.
